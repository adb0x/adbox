02

engine features

COMPONENT SYSTEM

Lua Component Architecture

Every game object is an Actor. Behavior comes from Lua table components with automatic lifecycle dispatch — OnStart, OnUpdate, OnLateUpdate, OnDestroy — managed by the C++ ComponentDB.

SCENE MANAGEMENT

Scene Loading & Transitions

Scenes defined in JSON. Actors load from reusable templates. DontDestroyOnLoad actors persist across scene changes with automated cleanup of scene-specific resources.

RENDERING

Sorted Sprite Renderer

Draw calls collected each frame and stable-sorted by layer and Y-position. Features camera zoom, screen-space UI rendering, and optimized texture caching via ImageDB.

PHYSICS

Box2D Integration

Full rigid body physics with fixtures, collision callbacks, and raycasting. The physics world steps precisely after actor updates to ensure state consistency before rendering.

EXECUTION LOOP

Component-Based Loop

A native C++ orchestration layer that manages precise execution order: Input processing, Lua lifecycle hooks (OnUpdate/OnLateUpdate), and C++ Particle System updates in a single deterministic tick.

EVENTS

Decoupled Event Bus

A Pub/Sub system for engine-wide communication. Subscriptions and unsubscriptions are deferred to the end of the frame to prevent collection modification errors during active event publication.

PARTICLES

High-Performance Particles

Built-in C++ ParticleSystem using Data-Oriented Design. Supports custom bursts, polar coordinate emission, and linear interpolation of scale/color over a particle's lifetime with free-list memory re-use.

INPUT

Unified Input Layer

Supports Keyboard, Mouse, and GameControllers (including raw Joystick axis mapping). Features frame-synced state management to handle "Just Pressed" vs "Held" logic across all devices.

03

Hardware Integration

NEW FEATURE

Full Controller &
Racing Wheel Support

Extended the SDL input layer to support SDL_GameController events alongside the existing keyboard and mouse system. Button states use a unified state machine for frame-perfect input with no polling overhead.

Implemented raw SDL_Joystick access to support specialized racing hardware. This allows for high-fidelity steering input and custom normalization of independent pedal and shifter axes that fall outside standard abstractions.

  • Logitech G29 Steering Wheel — High-resolution steering axis mapping
  • Integrated Triple-Pedal Set — Independent normalization for Gas, Brake, and Clutch
  • 6-Speed H-Pattern Shifter — Sequential and direct gear-state detection
  • Xbox & PlayStation Layouts — Automatic remapping and unified button indexing
  • Haptic Feedback — Native Rumble API exposed to Lua via Input.RumbleController()
