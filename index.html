02

engine features

COMPONENT SYSTEM

Lua Component Architecture

Every game object is an Actor. Behavior comes from Lua table components with automatic lifecycle dispatch — OnStart, OnUpdate, OnLateUpdate, OnDestroy — managed by the C++ ComponentDB.

SCENE MANAGEMENT

Scene Loading & Transitions

Scenes defined in JSON. Actors load from reusable templates. DontDestroyOnLoad actors persist across scene changes. Fade transitions handled by a dedicated SceneTransitionManager component.

RENDERING

Sorted Sprite Renderer

Draw calls collected each frame and sorted by layer and Y-position before submission. Camera zoom and position with smooth lerp. UI layer rendered independently of world space.

PHYSICS

Box2D Integration

Full rigid body physics with fixtures, collision callbacks, and raycasting exposed to Lua. Physics step runs after all actor updates each frame.

TURN-BASED LOGIC

Turn-Based Game Loop

A global ProcessTurn() function triggers AI movement, vision cone evaluation, and blocking map updates in a defined order — all written in Lua on top of the engine's frame loop.

EVENTS

Event Bus

Decoupled pub/sub event system accessible from Lua. Subscriptions processed at the end of each frame to avoid mid-frame side effects.

PARTICLES

Particle System

Data-driven particle emitters updated each frame. Config-driven spawn rate, lifetime, velocity, and color over lifetime.

INPUT

Unified Input Layer

Keyboard, mouse, and controller input unified behind a single Input namespace in Lua. GetKey / GetKeyDown / GetKeyUp mirrors across all device types with per-frame state flushing.

03

final project feature

NEW FEATURE

Full Controller &
Racing Wheel Support

Extended the SDL input layer to support SDL_GameController events alongside the existing keyboard and mouse system. Button states use the same INPUT_STATE enum (UP / JUST_BECAME_DOWN / DOWN / JUST_BECAME_UP) for frame-perfect input with no polling overhead.

Added raw SDL_Joystick access for devices that fall outside SDL's GameController abstraction — specifically to support the Logitech G29 steering wheel, whose pedals and shifter require direct joystick axis queries.

  • Xbox controller — full button, axis, and d-pad support
  • PlayStation layout controller — remapped button indices
  • Logitech G29 steering wheel — steering, gas, brake, clutch, shifter, and all face buttons
  • Hotplug — controllers connected after startup detected automatically
  • Rumble API exposed to Lua via Input.RumbleController()
